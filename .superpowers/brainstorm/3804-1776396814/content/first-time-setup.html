How does a new subscriber set their password?

When someone pays and visits access.html for the first time, no password is on file yet. What happens when they try to sign in?

A

Form transforms on submit

They fill in email + password and hit Sign in. The backend detects no password is set and the form switches to "Create your password" mode with a confirm field.

After first sign-in attempt
No password set yet — create one below to secure your account.
Subscription email
Create password
Confirm password
B

Email link to set password

When we detect no password on first sign-in, we send them a one-time setup link to their email. They click it, set their password, and are signed in. More secure, but adds email friction.

After first sign-in attempt
We've sent a password setup link to you@example.com.
Check your inbox to create your password.
Didn't get it? Check your spam folder or resend the link.
C

Separate onboarding page after payment

Stripe redirects new subscribers to a dedicated setup page (e.g. setup.html?email=...) immediately after checkout. They set their password before ever reaching access.html.

setup.html — post-checkout
Welcome to Text Boss!
Create a password to secure your account.
Create password
Confirm password

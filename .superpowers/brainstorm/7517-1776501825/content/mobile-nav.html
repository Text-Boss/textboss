A — Bottom Tab Bar

Persistent tabs at the bottom of the screen — the standard iOS/Android pattern. Always visible, thumb-friendly, instantly familiar. Each tab switches the full-screen view.

Pros

  • Most familiar to users
  • One-thumb navigation
  • Always visible

Cons

  • Takes permanent screen space
  • Max ~4-5 tabs

B — Swipe Between Sections

Full-screen sections you swipe left/right to navigate — like Stories or a card deck. A dot indicator shows your position. No permanent nav bar eats screen space.

Pros

  • Maximum screen space
  • Immersive, modern feel
  • Natural gesture pattern

Cons

  • Less discoverable
  • Hard to jump sections quickly

C — Top Header + Drawer

Hamburger menu top-left opens a slide-in drawer for section navigation. Thread tabs sit just below the header. Closer to the desktop experience — familiar if users already know the web app.

Pros

  • Familiar from desktop
  • Room for many sections

Cons

  • Menu hidden by default
  • Less thumb-friendly
  • Drawer pattern falling out of favor

My recommendation: A — Bottom Tab Bar — it's the native iOS pattern users expect and maps cleanly to Chat, Schedule, Threads, and Settings.
